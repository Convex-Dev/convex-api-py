get_account_info(address_account)¶

Get account information. This will only work with an account that has a balance or has had some transactions processed on the convex network. New accounts with no transfer or transactions will raise:

ConvexRequestError(404, ‘The Account for this Address does not exist.’) error

The returned information is dictionary of account information.

get_balance(address_account, account_from=None)¶

Get a balance of an account. At the moment the account needs to have a balance to get the balance of it’s account or any other account. Event though this is using a query request.

Parameters

  • str address_account (Account,) – Address or Account to get the funds for.

  • account_from (Account) – Optional account to use to make the request. This account should have a balance to make the request.

Returns

Return the current balance of the address or account address_account

query(transaction, address_account, language=None)¶

Run a query transaction on the block chain. Since this does not change the network state, and so the account does not need to sign the transaction. No funds will be used when executing this query.

Parameters

  • transaction (str) – Transaction to execute. This can only be a read only transaction.

  • str address_account (Account,) – Account or str address of an account to use for running this query.

Returns

Return the resultant query transaction

send(transaction, account, language=None, sequence_retry_count=20)¶

Send transaction code to the block chain node.

Parameters

  • transaction (str) – The transaction as a string to send

  • account (Account) – The account that needs to sign the message to send

  • language (str) – Language to use for this transaction. Defaults to LANGUAGE_LISP.

  • sequence_retry_count (int) – Number of retries to do if a SEQUENCE error occurs. When sending multiple send requsts on the same account, you can get SEQUENCE errors, This send method will automatically retry again

Returns

The dict returned from the result of the sent transaction.
